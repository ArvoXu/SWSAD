## 📋 項目概述

**智慧倉儲與銷售分析系統（Smart Warehousing and Sales Analysis Dashboard）** 是一套完整的企業級數據管理和分析解決方案，為零售連鎖企業（樂雅樂快餐）提供實時的庫存監控、銷售分析和補貨策略優化功能。

### 核心價值主張
- **實時數據自動抓取**：通過 Web 爬蟲定時獲取多個門店的庫存和銷售數據
- **多維度分析儀表板**：提供庫存、銷售、補貨三大核心模塊
- **智能補貨推薦**：基於銷售趨勢和庫存水位的補貨策略建議
- **用戶分級管理**：支持超級管理員、營運主管和加盟主不同的訪問權限
- **雲端部署**：支持容器化部署和 PostgreSQL 數據庫集成

## 🎯 核心功能模塊

### 1️⃣ **庫存管理模塊** (`scraper.py`, `inventory_tab`)
**功能特性：**
- 自動登入樂雅樂管理系統（Yokai Express Manager）
- 多頁面分頁爬取全部門店庫存數據
- 實時數據解析和結構化存儲
- 支持本地 JSON 文件上傳和批量導入

**技術亮點：**
- Selenium 無頭瀏覽器運行
- 動態等待和容錯機制
- 支持多倉庫庫存管理（Excel 文件上傳）
- 庫存警告閾值設定

**數據模型：**
```
Inventory (庫存)
- store_name (門店名)
- machine_id (機台ID)
- product_name (產品名)
- quantity (庫存量)
- last_updated (更新時間)

Warehouse (倉庫)
- warehouse_name
- product_name
- quantity
- updated_at
```

### 2️⃣ **銷售分析模塊** (`salesscraper.py`, `sales_tab`)
**功能特性：**
- 自動下載銷售報告 Excel 文件
- 按時間範圍的銷售趨勢分析
- 按門店/產品的銷售排名
- 銷售預測和環比分析

**技術亮點：**
- 自動化下載流程（無需手動操作）
- OpenPyXL 解析 Excel 數據
- 實時數據更新
- Chart.js 動態圖表展示

**數據模型：**
```
Transaction (銷售交易)
- store_name
- product_name
- quantity_sold
- sale_amount
- transaction_date
- timestamp
```

### 3️⃣ **補貨策略模塊** (`replenishment_tab`)
**功能特性：**
- 基於銷售速率的智能補貨建議
- 庫存低位警告
- 補貨優先級排序
- 補貨執行歷史記錄

**算法邏輯：**
- 計算日均銷售量 (Daily Average Sales)
- 計算安全庫存水位 (Safety Stock Level)
- 補貨數量 = 最大庫存 - 當前庫存
- 優先級 = 缺貨風險度 × 銷售額貢獻度

**技術實現：**
```javascript
// 補貨計算引擎
replenishmentScore = (avgDailySales * leadTime) + safetyStock
priority = (maxStock - currentStock) / avgDailySales
```

### 4️⃣ **用戶管理與權限控制**
**功能特性：**
- 三層用戶角色系統：
  - 超級管理員（全部數據訪問）
  - 營運主管（門店級數據）
  - 加盟主（自有門店數據）
- 登入驗證和會話管理
- 用戶設定和偏好保存

**技術實現：**
- Flask 會話管理
- Werkzeug 密碼哈希加密
- 基於角色的訪問控制 (RBAC)

**數據模型：**
```
User (用戶)
- username
- password_hash
- role (admin, manager, franchisee)
- assigned_stores (加盟主可見門店)
- created_at
- last_login
```

### 5️⃣ **數據導入與導出**
**功能特性：**
- JSON 文件上傳（爬蟲輸出結果）
- Excel 批量導入（倉庫數據）
- 儀表板截圖導出（html2canvas）
- 數據驗證和錯誤報告

**支持的文件格式：**
```json
{
  "store": "門店名",
  "machine_id": "機台ID",
  "product_name": "產品名",
  "quantity": 100,
  "last_updated": "2025-01-01T12:00:00+08:00",
  "process_time": "2025-01-01T12:00:00+08:00"
}
```

### 6️⃣ **定時任務與通知**
**功能特性：**
- 定時爬蟲任務調度（支持多倉庫）
- 庫存警告通知（當庫存 < 閾值）
- 更新日誌記錄（UpdateLog）
- 通知已發送狀態追蹤

**數據模型：**
```
NotificationSent (通知記錄)
- notification_type
- target_store
- product_name
- sent_at
- acknowledged_by

UpdateLog (更新日誌)
- log_type (inventory, sales, warehouse)
- record_count
- status (success, error, partial)
- timestamp
```

### 典型使用場景
**場景 1：營運主管查看實時庫存**
1. 營運主管登入 presentation-login.html
2. 系統驗證用戶權限 (RBAC)
3. 前端請求 `/api/inventory` 端點
4. Flask 從數據庫查詢當前用戶可見的門店數據
5. Chart.js 渲染即時庫存圖表

**場景 2：自動補貨推薦**
1. 定時任務每日執行 (schedule)
2. 查詢近 7 天銷售數據
3. 計算 DailyAvgSales 和 SafetyStock
4. 生成補貨清單並通知相關人員
5. 記錄到 UpdateLog

**場景 3：上傳門店庫存數據**
1. 在本地運行 `local_scraper.py`
2. 生成 `inventory_upload_YYYYMMDD_HHMMSS.json`
3. 通過前端上傳至 `/upload-inventory-file`
4. Flask 驗證 JSON 結構
5. SQLAlchemy 批量插入 Inventory 表
6. 前端自動刷新儀表板

### 3. 定時任務調度
```python
# 使用 Schedule 庫進行簡單可靠的任務調度
schedule.every().day.at("02:00").do(run_scraper)
schedule.every(6).hours.do(run_sales_scraper)

# 後台線程運行調度器
def run_scheduler():
    while True:
        schedule.run_pending()
        time.sleep(60)

scheduler_thread = threading.Thread(target=run_scheduler, daemon=True)
scheduler_thread.start()
```
